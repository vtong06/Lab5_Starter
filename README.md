1. I would not use a unit test to test a message feature of a messaging application because there isn't really a way to test it with unit tests. Messaging in an application involves many components such as the input, how its displayed, etc. Units tests can't test how these components interact with each other so I would not be able to use a unit test
2. I would use a unit test to test the max message length because we would easily be able to test it by passing inputs of different lengths and checking whether they’re accepted or rejected. If for example, the max length would be 1000 characters, we would just have to find the length of our input and see if it exceeds 1000.
